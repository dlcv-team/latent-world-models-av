Analysis
--------

Issue 1: EMA weights are broken
  - EMA decay=0.9999 with only ~7800 training steps
  - Effective EMA window is ~10K steps, so initial (random) weights
    still contribute ~46% to the EMA average
  - noise_MSE with EMA: 0.80-0.95 (barely better than zero-prediction baseline of 1.0)
  - noise_MSE with main: 0.007-0.033 (50-100x better)
  - Fix: use lower decay (0.999) or add EMA warmup, or just use main weights

Issue 2: Data scale mismatch (affects both EMA and main)
  - Adapted embeddings: per-element std = 0.065, L2 norm = 1.28 (384-d)
  - Diffusion noise: per-element std = 1.0
  - At t=100 (alpha_bar=0.975): SNR per element = 0.065*0.987 / (1.0*0.158) = 0.41
  - At t=500 (alpha_bar=0.50):  SNR per element = 0.065*0.707 / (1.0*0.707) = 0.065
  - Model can reconstruct at t=100 (CosSim=0.88) but not at t=500+ (CosSim=0.50)
  - DDIM starts from t=980 (pure noise) where SNR << 0.01, so generation fails
  - Fix: normalize adapted embeddings to unit variance before diffusion training

Key insight: the adapter nn.Linear(1024, 384, bias=False) was trained jointly with
the DiT, but nothing enforced that its output would have reasonable variance for
the cosine diffusion schedule. The cosine schedule assumes data ~ N(0, ~1).
