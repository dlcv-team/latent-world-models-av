# DeltaCosSim results summary — `vjepa2_rep64` (C5)

**CosSim values at k=1..4.** Evaluated against `vjepa2_rep64` on the 5,419-sequence test split, the conditioned predictor achieves CosSim_cond(k=1)=0.9955, CosSim_cond(k=2)=0.9955, CosSim_cond(k=3)=0.9955, CosSim_cond(k=4)=0.9955, while the unconditional baseline (actions zeroed out) achieves CosSim_uncond(k=1)=0.9999, CosSim_uncond(k=2)=0.9999, CosSim_uncond(k=3)=0.9999, CosSim_uncond(k=4)=0.9999. CosSim_conditioned is **not** monotonically decreasing in k, which is unusual for an autoregressive latent predictor and deserves a second look from M1.

**Does action conditioning help?** Per-horizon deltas are Δ(k=1)=-0.0044, Δ(k=2)=-0.0044, Δ(k=3)=-0.0044, Δ(k=4)=-0.0044. DeltaCosSim is non-positive at every horizon (best at k=3, Δ=-0.0044; horizon-mean Δ=-0.0044). The current latent predictor pipeline does not surface a benefit from action conditioning -- on this evidence the action input is being ignored or actively hurting prediction.

**Comparison to BC baseline.** On the same encoder, the BC baseline reports test RMSE = 0.0774 (± 0.0009 over 3 seeds) on the 6019-sample test split — i.e. a non-trivial linear-decoder mapping from `z_t` to (steer, accel) does exist. However, with DeltaCosSim non-positive at every horizon and CosSim_uncond essentially matching CosSim_cond (mean ≈ 0.9955), the current evidence does **not** support the claim that the latent predictor provides a richer representation than BC: actions add no measurable signal beyond what `z_t` already encodes, so the predictor is effectively learning an identity / smoothing operator. The honest read is that the *encoder* captures decodable action-relevant features at time t (BC works), but the *trained predictor* in this run does not exploit them to model action-driven dynamics.
